# audio-hotkeys PRD

## 1. 개요

| 항목 | 내용 |
|------|------|
| 제품명 | audio-hotkeys |
| 한 줄 요약 | Windows에서 `Ctrl+Alt+NumPad 0–9`로 오디오 입·출력 장치와 볼륨 스냅샷을 즉시 전환하는 경량 트레이 앱 |
| 플랫폼 | Windows 10 / 11 |
| 저장소 | https://github.com/kuroicode-beep/audio-hotkeys |
| 문서 버전 | 1.0 |
| 작성일 | 2026-07-13 |

## 2. 배경 / 문제

오디오 장치·볼륨 조합을 자주 바꾸는 환경(헤드셋 ↔ 스피커, USB 인터페이스, 마이크 전환 등)에서 Windows 기본 UI는 단계가 많고 반복 비용이 크다.

사용자는 **미리 저장해 둔 조합을 고정 단축키 한 번으로 복원**하고 싶다. 기능 범위는 최소로 유지하고, UI는 다크 모드·고대비·가벼운 상주 트레이 앱이어야 한다.

## 3. 목표

### 3.1 제품 목표

1. NumPad 슬롯 0–9에 오디오 스냅샷을 저장·적용한다.
2. 단축키는 **고정**한다: `Ctrl + Alt + NumPad 0` … `NumPad 9`.
3. 트레이에서 설정·슬롯 적용·종료가 가능하다.
4. 설정 UI와 트레이 메뉴는 **다크 모드**를 강제한다.
5. 의존성과 상주 리소스를 최소화한다.

### 3.2 비목표 (이번 버전에서 하지 않음)

- 단축키 커스터마이즈
- 앱별(프로세스별) 오디오 라우팅
- 이퀄라이저 / 공간 음향 / 샘플레이트 설정
- macOS / Linux 지원
- 클라우드 동기화
- 플러그인·확장 시스템

## 4. 사용자 / 시나리오

### 4.1 주요 사용자

- 여러 출력·입력 장치를 하루에도 여러 번 바꾸는 PC 사용자
- 스트리밍·통화·음악 청취 프로필을 키로 전환하고 싶은 사용자

### 4.2 핵심 시나리오

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| S1 | 설정에서 슬롯 1에 헤드셋 출력 + 헤드셋 마이크 + 볼륨을 저장 | `config.json`에 슬롯 1이 저장됨 |
| S2 | `Ctrl+Alt+NumPad1` 입력 | 슬롯 1의 장치·볼륨이 적용되고 토스트로 요약 표시 |
| S3 | `(unchanged)`로 출력만 지정한 슬롯 적용 | 입력/볼륨은 유지되고 출력만 변경 |
| S4 | 트레이 우클릭 → 슬롯 선택 | 단축키와 동일하게 스냅샷 적용 |
| S5 | Capture current | 현재 기본 입·출력·볼륨이 폼에 채워짐 |

## 5. 기능 요구사항

### 5.1 스냅샷 모델

각 슬롯(0–9)은 다음 필드를 가진다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `name` | string | 표시 이름 |
| `output_id` | string | 시스템 출력 장치 ID. 빈 값 = 변경 안 함 |
| `output_name` | string | 위 장치의 표시 이름. ID가 무효일 때 재매칭 키 (v1.1.0) |
| `input_id` | string | 시스템 입력 장치 ID. 빈 값 = 변경 안 함 |
| `input_name` | string | 위 장치의 표시 이름. ID가 무효일 때 재매칭 키 (v1.1.0) |
| `output_volume` | int \| null | 시스템 출력 볼륨 0–100. `null` = 변경 안 함 |
| `input_volume` | int \| null | 시스템 입력 볼륨 0–100. `null` = 변경 안 함 |
| `kakao_output_id` | string | 카카오톡 전용 출력 장치. 빈 값 = 변경 안 함 |
| `kakao_output_name` | string | 위 장치의 표시 이름. 재매칭 키 (v1.1.0) |
| `kakao_input_id` | string | 카카오톡 전용 입력 장치. 빈 값 = 변경 안 함 |
| `kakao_input_name` | string | 위 장치의 표시 이름. 재매칭 키 (v1.1.0) |
| `kakao_output_volume` | int \| null | 카카오톡 앱 볼륨(우선). `null` = 변경 안 함 |
| `kakao_input_volume` | int \| null | 예비 앱 볼륨 필드(출력 볼륨이 없을 때 사용) |

카카오톡 필드는 `KakaoTalk.exe`가 실행 중일 때만 적용된다. 장치 라우팅은 번들된 `svcl.exe`(Nir Soft)를 사용한다.

### 5.2 단축키 (고정)

| 단축키 | 동작 |
|--------|------|
| `Ctrl+Alt+NumPad0` … `Ctrl+Alt+NumPad9` | 해당 슬롯 스냅샷 적용 |
| `Ctrl+Alt+Shift+NumPad0` … `NumPad9` | **현재 오디오 상태를 해당 슬롯에 저장** (v1.2.0) |
| `Ctrl+Alt+.` | **직전에 적용한 슬롯으로 토글** (v1.3.0) |

**빠른 저장 (v1.2.0)** — 설정 창을 열지 않고 현재 시스템 장치·볼륨을 슬롯에 기록한다.
슬롯 이름(`name`)은 보존한다. 장치 ID와 **이름**을 함께 저장하므로 이후 재매칭이 보장된다.
KakaoTalk이 실행 중이면 함께 캡처하고, 미실행이면 해당 슬롯의 기존 KakaoTalk 필드를 **지우지 않는다**
(빈 캡처로 덮어쓰지 않음). 결과는 초록색 OSD(`✓ 저장됨`) + 요약 토스트로 알린다.

**직전 슬롯 토글 (v1.3.0)** — 마지막으로 적용한 슬롯과 그 직전 슬롯을 한 키로 왕복한다.
적용 경로(단축키·트레이 메뉴 공통)가 성공했을 때만 기록하며, 같은 슬롯을 연속 적용해도 직전 기록은
덮이지 않는다. NumPad `.`(VK_DECIMAL)와 메인 키보드 `.`(VK_OEM_PERIOD)를 모두 등록하므로
NumLock이 꺼져 있어도 메인 키보드 쪽은 동작한다. 기록은 **메모리에만** 두므로 재시작 직후에는
슬롯을 두 개 적용해야 활성화된다(그 전에는 안내 토스트).

- NumLock ON 상태를 전제로 한다.
- 단축키 변경 UI는 제공하지 않는다.
- 적용 시 OSD/토스트로 슬롯 이름과 변경 요약을 짧게 표시한다.

### 5.3 설정 창

- 다크 배경, 고대비 텍스트, 최소 글자 크기 16px, 포커스 링 `#FFFF00`.
- 슬롯 선택 후 아래 **리스트(콤보박스)** 만 제공한다.
  - 출력장치
  - 입력장치
  - 출력볼륨
  - 입력볼륨
- 각 리스트 첫 항목은 `(unchanged)`.
- 볼륨 리스트는 `0, 5, 10, …, 100` 단위.
- 액션
  - **Save slot**: 현재 폼을 해당 슬롯에 저장
  - **Apply now**: 저장 없이 폼 내용 즉시 적용
  - **Capture current**: 현재 기본 장치·볼륨을 폼에 반영(볼륨은 5단위로 반올림)
  - **Close**: 창 닫기

### 5.4 트레이

| 입력 | 동작 |
|------|------|
| 좌클릭 | 설정 창 열기 |
| 우클릭 | 다크 커스텀 메뉴 |
| 메뉴 — Settings | 설정 창 |
| 메뉴 — 슬롯 목록 | 해당 슬롯 적용 |
| 메뉴 — Quit | 핫키 해제 후 종료 |

네이티브 Windows 트레이 메뉴 대신 **커스텀 다크 팝업**을 사용한다.

### 5.5 적용 순서

1. 출력 장치가 지정되면 기본 출력으로 설정
2. 입력 장치가 지정되면 기본 입력으로 설정
3. 출력 볼륨이 지정되면 **전환 후 기본 출력**에 볼륨 적용(뮤트 해제)
4. 입력 볼륨이 지정되면 **전환 후 기본 입력**에 볼륨 적용(뮤트 해제)

빈 스냅샷(모든 필드 unchanged)을 적용하면 “empty” 요약을 표시하고 시스템을 변경하지 않는다.

**부분 실패 규칙 (v1.1.0)** — 각 필드는 독립적으로 적용된다. 한 장치가 사라졌거나 COM 호출이
실패해도 예외를 위로 던지지 않고 그 필드만 건너뛴 뒤 나머지를 계속 적용한다. `apply_snapshot()`은
`ApplyResult(summary, warnings)`를 돌려주며, 프로필 OSD는 **항상** 표시되고(단축키의 존재 이유)
경고 상세는 별도 토스트로 분리한다. v1.0.0은 첫 실패에서 전체를 중단하고 OSD 대신 원시 HRESULT
에러(`Apply failed: (-2147023728, '요소가 없습니다.')`)를 띄웠다.

## 6. 비기능 요구사항

| 항목 | 요구 |
|------|------|
| 경량성 | 트레이 상주, 불필요한 백그라운드 폴링 없음. 핫키는 Win32 `RegisterHotKey` |
| UI | 다크 테마 기본, WCAG 대비 목표(본문 ≥ 4.5:1), 클릭 타겟 여유 확보 |
| 설정 저장 | `%LOCALAPPDATA%\audio-hotkeys\config.json` |
| 의존성 | `pycaw`, `comtypes`, `pystray`, `Pillow`, 표준 `tkinter` |
| 라이선스 | MIT |
| 공개 | GitHub public 저장소 |

## 7. UX / UI 원칙

1. 첫 실행부터 “단축키는 고정”임을 설정 창에 명시한다.
2. 한 화면에서 슬롯 하나만을 편집한다(복잡 대시보드 금지).
3. 색만으로 상태를 구분하지 않고 텍스트 라벨을 병행한다.
4. 적용 결과는 토스트로만 짧게 알리고, 모달 확인창은 쓰지 않는다.

## 8. 기술 개요

| 영역 | 구현 |
|------|------|
| 언어 | Python 3.11+ |
| 기본 장치 전환 | Windows PolicyConfig COM (`SetDefaultEndpoint`) |
| 장치 열거 / 볼륨 | `pycaw` (WASAPI) |
| 전역 핫키 | `RegisterHotKey` + 메시지 루프 스레드 |
| 트레이 | `pystray` + 우클릭 시 커스텀 다크 메뉴 |
| 설정 UI | `tkinter` 다크 스타일 |

패키지 진입점: `python -m audio_hotkeys` / `run.bat`

## 9. 수용 기준 (Acceptance Criteria)

- [ ] `Ctrl+Alt+NumPad0–9`가 등록되고, 다른 앱이 포그라운드여도 동작한다.
- [ ] 설정에서 네 가지 필드만 리스트로 선택·저장된다.
- [ ] `(unchanged)` 필드는 적용 시 건드리지 않는다.
- [ ] 트레이 우클릭 메뉴가 다크 테마로 표시된다.
- [ ] 설정 창이 다크 테마·고대비로 표시된다.
- [ ] 적용 시 토스트에 슬롯명과 변경 요약이 보인다.
- [ ] 설정이 `%LOCALAPPDATA%\audio-hotkeys\config.json`에 유지된다.
- [ ] `run.bat`으로 트레이 실행이 가능하다.

## 10. 리스크 / 제약

| 리스크 | 영향 | 완화 |
|--------|------|------|
| PolicyConfig는 비문서화 COM | Windows 업데이트 후 동작 변화 가능 | 최소 호출면 유지, 회귀 시 대체 CLI 검토 |
| NumPad / NumLock | NumLock OFF면 NumPad가 다른 VK를 보내 핫키 자체가 뜨지 않음 | v1.1.0 — 시작 시 NumLock 상태를 감지해 토스트로 경고 |
| 핫키 충돌 | 다른 앱이 동일 조합 선점 시 등록 실패 | v1.1.0 — 슬롯별 개별 등록(하나 실패해도 나머지 유지) + 실패 슬롯·사유 토스트 |
| 장치 ID 변경 | USB 재연결 등으로 ID가 바뀌면 슬롯이 무효 | v1.1.0 — 스냅샷이 이름을 함께 저장해 자동 재매칭, 실패 필드만 건너뜀 |
